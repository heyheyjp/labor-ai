Naming conventions

Table names: singular, snake_case
occupation_task not occupation_tasks — follows the original relational model; a table names a type, not a collection
Be explicit — no ambiguous abbreviations
detailed_occupation_code not soc_code; major_group_name not major_group; soc_detailed_occupation_id not occupation_id
Currency: append _usd to mean and median wage fields only
annual_mean_wage_usd, annual_median_wage_usd — percentile fields omit _usd as currency is established by context
Wage time period: always explicit
annual_mean_wage_usd not mean_wage
Percentile fields: p{N} format
annual_wage_p10, annual_wage_p90 — distinct from pct which is reserved for percentage
Percentage fields: _pct suffix
tasks_automatable_pct — never use p{N} format for percentages
Boolean fields: is_ prefix
is_automatable not automatable
Timestamps (datetime with time zone): _utc suffix, always stored in UTC
created_at_utc, updated_at_utc on every table
Dates (calendar date, no time component): no _utc suffix
published_date — a date has no timezone
Surrogate PKs: UUID on all tables that need them, field named id
Used where no meaningful natural key exists, or where the natural key alone can't guarantee uniqueness across data versions (e.g. SOC revision cycles)
FK field names: {table_name}_id for UUID surrogate FKs
e.g. soc_detailed_occupation_id on operational tables, soc_major_group_id on soc_minor_group — the _id suffix signals a UUID FK. String natural keys use descriptive names like detailed_occupation_code.
SOC Taxonomy Reference Tables — loaded once from BLS 2018 SOC Structure file, insertion order: major → minor → broad → detailed
soc_major_group
Top level of the SOC taxonomy. 23 groups.
Source: BLS 2018 SOC Structure
FieldTypeSourceNotes
idPK UUID generated Surrogate key. Necessary because major_group_code alone is not revision-safe as a PK — codes can change between SOC revisions.
major_group_codereq str BLS SOC e.g. "15-0000". Last four digits are always zeros at this level.
major_group_namereq str BLS SOC e.g. "Computer and Mathematical Occupations".
revision_yearreq int metadata SOC revision this row belongs to. Value: 2018 for all current data. Enables future coexistence of multiple SOC revisions without schema changes.
created_at_utcauto datetime generated Auto-set to current UTC on insert.
updated_at_utcauto datetime generated Auto-set on insert and on every update.
Constraints
  • UNIQUE (major_group_code, revision_year)

soc_detailed_occupation
Leaf level of the SOC taxonomy. ~830 occupations. All operational tables reference this one.
Source: BLS 2018 SOC Structure + O*NET descriptions

The grain at which all employment, exposure, projection, and task data is recorded. Serves the role of an occupation table for V1. A separate app-level occupation table may be introduced later if app-specific fields are needed.

FieldTypeSourceNotes
idPK UUID generated Surrogate key. All operational tables FK to this.
soc_broad_occupation_idFK UUID derived References soc_broad_occupation.id. Enables traversal up the full hierarchy via joins.
detailed_occupation_codereq str BLS SOC e.g. "15-1252". The SOC code shared across OEWS, Pew, O*NET, and BLS projections — the natural join key across all source datasets.
detailed_occupation_namereq str BLS SOC e.g. "Software Developers". Official title, used in Claude's responses.
descriptionopt str | null O*NET Plain English description of what workers do. From O*NET, not BLS. e.g. "Develop general computer applications software or specialized utility programs."
revision_yearreq int metadata Value: 2018 for all current data. SOC codes can change between revisions — this records which revision a code belongs to.
created_at_utcauto datetime generated Auto-set on insert.
updated_at_utcauto datetime generated Auto-set on insert and update.
Constraints
  • UNIQUE (detailed_occupation_code, revision_year)
Operational Tables — populated by ingestion scripts
occupation_geo_estimate
Employment counts and wage estimates by occupation and geography. One row per occupation × geography × year.
Source: BLS OEWS (annual)

BLS surveys ~1.1 million employers annually. These are estimates derived from a survey, not a census — hence the table name. BLS suppresses values where sample size is too small, so many fields are nullable. Geography is stored inline (not normalized) — geo entities carry no independent attributes worth a separate table at this stage.

FieldTypeSourceNotes
idPK UUID generated Surrogate key. Logical key is the composite unique constraint below.
soc_detailed_occupation_idFK str BLS OEWS References soc_detailed_occupation. Aggregate/group-level rows (codes ending in 0, 00, 0000) are filtered out during ingestion — only detailed occupation rows are loaded.
survey_yearreq int BLS OEWS Year of the OEWS survey. Latest available: 2024. Storing year enables multi-year accumulation for trend queries without schema changes.
geo_typereq str derived "national" | "state" | "metro". BLS publishes three separate files per year — this field merges them into one table.
geo_nameopt str | null BLS OEWS Human-readable geography. e.g. "California" or "San Francisco-Oakland-Hayward, CA". Null for national rows.
geo_codeopt str | null BLS OEWS Machine-readable geography code. FIPS for states (e.g. "06" = California), MSA code for metro areas. Null for national. Note: state FIPS and MSA codes come from different numbering systems and are not globally unique — always filter by geo_type alongside geo_code.
est_worker_countopt int | null BLS OEWS Estimated number of employed workers in this occupation and geography. Nullable — BLS suppresses small-sample estimates.
annual_mean_wage_usdopt float | null BLS OEWS Mean annual wage. Can be skewed upward by top earners — median is usually more representative.
annual_median_wage_usdopt float | null BLS OEWS Median annual wage (50th percentile). Most commonly cited figure. "Half of workers in this occupation earn more than $X."
annual_wage_p10opt float | null BLS OEWS 10th percentile annual wage. Proxy for entry-level / early-career compensation floor.
annual_wage_p25opt float | null BLS OEWS 25th percentile annual wage.
annual_wage_p75opt float | null BLS OEWS 75th percentile annual wage.
annual_wage_p90opt float | null BLS OEWS 90th percentile annual wage. Together the four percentile columns let a user roughly self-locate in their occupation's wage distribution.
created_at_utcauto datetime generated Auto-set on insert.
updated_at_utcauto datetime generated Auto-set on insert and update.
Constraints
  • UNIQUE (soc_detailed_occupation_id, geo_type, geo_code, survey_year)
Notes
  • Geo fields are inline — no separate reference table. Geography has no rich independent attributes to warrant normalization at this stage.
  • Ingestion is append-only by year — old year rows are retained, enabling trend queries over time.
  • BLS also publishes hourly wage figures in the same files — not included in V1 but naming would follow hourly_mean_wage_usd etc. if added later.
occupation_ai_exposure
AI automation exposure scores per occupation. One row per occupation per scoring methodology.
Source: Pew Research Center (derived from O*NET task data)

Pew evaluated each O*NET work activity for AI automatability and rolled scores up to the occupation level. The surrogate PK + unique constraint on (code, source) allows future methodologies (Goldman, OECD, etc.) to coexist as additional rows rather than replacing existing data.

FieldTypeSourceNotes
idPK UUID generated Surrogate key. Logical key is (soc_detailed_occupation_id, exposure_score_source).
soc_detailed_occupation_idFK str Pew / O*NET References soc_detailed_occupation.
exposure_score_sourcereq str metadata Identifies the methodology that produced the score. Value: "pew_2023" for all current data. Part of the unique constraint — allows multiple methodologies per occupation in future.
exposure_scorereq float Pew Normalized 0.0–1.0. Higher = more exposed. e.g. Data Entry Keyers ≈ 0.9, EMTs ≈ 0.15.
exposure_tierreq str derived "high" | "medium" | "low". Bucketed from exposure_score at ingestion time.
tasks_automatable_pctreq float Pew / O*NET Percentage of this occupation's tasks Pew classified as highly automatable. e.g. 0.72 = 72% of tasks. More interpretable than the normalized score for end users.
exposure_summaryopt str | null generated Plain English explanation of why this occupation has this score. Generated once via Claude API after initial ingestion and stored permanently. Null until generation script is run.
created_at_utcauto datetime generated Auto-set on insert.
updated_at_utcauto datetime generated Auto-set on insert and update.
Constraints
  • UNIQUE (soc_detailed_occupation_id, exposure_score_source)
occupation_projection
10-year employment outlook per occupation. One row per occupation per projection cycle.
Source: BLS Employment Projections 2023–2033 (published every 2 years)

National-only — BLS does not publish projections at state or metro level. Storing base_year and projection_year allows future cycles to coexist as new rows rather than replacing current data.

FieldTypeSourceNotes
idPK UUID generated Surrogate key. Logical key is (soc_detailed_occupation_id, base_year).
soc_detailed_occupation_idFK str BLS References soc_detailed_occupation.
base_yearreq int BLS Starting year of the projection cycle. 2023 for current data.
projection_yearreq int BLS Ending year of the projection cycle. 2033 for current data.
base_employmentopt int | null BLS Employment count in the base year. Denominator for percentage change calculations.
projected_employmentopt int | null BLS Projected employment count in projection_year.
change_absopt int | null BLS Projected absolute change in number of jobs. Negative = shrinking field. Provided directly by BLS.
change_pctopt float | null BLS Percentage change in employment. e.g. 14.2 or -8.1. Most intuitive figure for end users.
growth_categoryopt str | null BLS BLS plain-English label. e.g. "Much faster than average" | "Faster than average" | "As fast as average" | "Slower than average" | "Decline". Provided directly by BLS.
created_at_utcauto datetime generated Auto-set on insert.
updated_at_utcauto datetime generated Auto-set on insert and update.
Constraints
  • UNIQUE (soc_detailed_occupation_id, base_year)
occupation_task
Individual work tasks per occupation, with AI automation risk per task. ~10–30 rows per occupation.
Source: O*NET task database + Pew automation classifications

Where occupation_ai_exposure gives a single score per occupation, this table provides the underlying task-level breakdown — which specific activities within the job are automatable. Enables "which parts of my job are most at risk?" queries.

FieldTypeSourceNotes
idPK UUID generated Surrogate key.
soc_detailed_occupation_idFK str O*NET References soc_detailed_occupation. Many-to-one — each occupation has many tasks.
task_descriptionreq str O*NET The actual task text. e.g. "Write code to create applications or programs." Directly surfaceable to users.
task_typereq str O*NET "core" = performed by most workers in the occupation. "supplemental" = performed by some. Core tasks are most relevant for exposure analysis.
is_automatablereq bool Pew Pew's binary classification — is this task highly exposed to AI automation?
automation_risk_levelreq str derived More granular than the boolean: "high" | "medium" | "low". Derived from Pew's underlying scores where available.
importance_scoreopt float | null O*NET O*NET's importance rating for this task, 0–5 scale. Higher = more central to the job. Enables weighting exposure by task importance rather than just counting tasks.
created_at_utcauto datetime generated Auto-set on insert.
updated_at_utcauto datetime generated Auto-set on insert and update.
research_source
One row per source document in the RAG corpus. Parent of research_chunk.
Source: manually curated — WEF, Pew, BLS, journalism

Holds all document-level metadata. The deduplication check on ingestion happens here — look up source_url before processing a document. If it already exists, skip. Not occupation-specific — any labor market or AI displacement research belongs here.

FieldTypeSourceNotes
idPK UUID generated Surrogate key. Referenced by research_chunk.research_source_id.
source_urlreq str metadata URL of the original document. The deduplication key — unique constraint prevents the same document being ingested twice.
source_titlereq str metadata Human-readable name of the document. e.g. "WEF Future of Jobs Report 2025". Surfaced to the user when Claude cites a source.
source_typereq str metadata "report" | "article" | "paper" | "government". Allows filtering retrieval by source type.
published_dateopt date | null metadata Publication date of the document. Plain date type — no time component, no timezone suffix. Null if unknown. Used to tell Claude how recent a source is.
created_at_utcauto datetime generated Auto-set on insert.
updated_at_utcauto datetime generated Auto-set on insert and update.
Constraints
  • UNIQUE (source_url)
research_chunk
Chunked text with vector embeddings. Child of research_source. The RAG retrieval unit.
Source: chunked + embedded at ingestion time

Each source document is split into overlapping chunks (~500 tokens, ~100 token overlap), embedded via Voyage AI, and stored here. At query time the user's question is embedded and the most similar chunks are retrieved via cosine similarity and fed to Claude.

FieldTypeSourceNotes
idPK UUID generated Surrogate key.
research_source_idFK UUID generated References research_source.id. All chunks for a document share the same research_source_id.
chunk_indexreq int generated 0-indexed position of this chunk within its source document. Allows reconstructing document order and fetching adjacent chunks when more context is needed.
chunk_textreq str ingestion The actual text content (~400–600 tokens). Overlaps slightly with adjacent chunks to avoid cutting ideas mid-sentence. This is what Claude reads when a chunk is retrieved.
embeddingopt vector(1024) Voyage AI 1024-dimensional float vector from Voyage AI's voyage-3 model. Stored in pgvector's native Vector column. Used for cosine similarity search. Null until embedding script is run. Dimension is model-specific — document in a comment if the model changes.
created_at_utcauto datetime generated Auto-set on insert.
updated_at_utcauto datetime generated Auto-set on insert and update.
Constraints
  • UNIQUE (research_source_id, chunk_index)

Key joins — canonical questions → tables

"How exposed to AI is my job?"
soc_detailed_occupation ⟶ occupation_ai_exposure
"Is my field projected to grow or shrink?"
soc_detailed_occupation ⟶ occupation_projection
"Which specific tasks in my job are most at risk?"
soc_detailed_occupation ⟶ occupation_task (is_automatable = true)
"How exposed is my occupation in California?"
occupation_ai_exposure + occupation_geo_estimate (geo_type='state', geo_name='California')
"Which jobs similar to mine have lower exposure?"
soc_detailed_occupation (via soc_broad_occupation_id hierarchy) ⟶ occupation_ai_exposure ORDER BY exposure_score ASC
"Which exposed jobs are growing anyway?"
occupation_ai_exposure (exposure_tier='high') ⟶ occupation_projection (change_pct > 0)
"What do researchers say about AI risk in my field?"
research_chunk (vector similarity search) ⟶ research_source (title, url, date for citation)
"Where do I fall in my occupation's wage distribution?"
occupation_geo_estimate (geo_type='national') wage percentile columns
