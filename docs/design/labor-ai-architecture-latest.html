Component Reference
Component Service / Tech Notes
Frontend Next.js · Vercel React/TypeScript app deployed on Vercel edge. Occupation search UI, results rendering, follow-up input. Auth.js handles GitHub OAuth session. Sentry Browser SDK for JS errors and Web Vitals.
Auth GitHub OAuth · Auth.js read:user scope only — GitHub username is the identity token. No email, no repo access. JWT session stored client-side. Vercel edge middleware handles IP-level rate limiting before requests reach the app.
API Server FastAPI · Python · Railway REST API: validates JWT, checks Upstash rate limit, fetches structured data from Supabase, runs vector search, calls Claude, returns synthesized response. Sentry Python SDK tracks endpoint latency and errors. Hobby plan ~$10–15/mo.
Rate Limiting Upstash Redis Per-user query counters keyed by GitHub user ID. TTL resets hourly. FastAPI returns 429 when limit exceeded. Serverless Redis — no persistent connection overhead. Free tier sufficient at demo traffic.
LLM Anthropic Claude Sonnet Receives structured context (SOC data, Pew score, retrieved research chunks) and returns synthesized prose with inline citation markers. Tool call pattern: Claude decides whether to query structured data or vector store. Called only on cache miss.
Primary DB Supabase · Postgres Stores full SOC taxonomy (4 levels), occupation_geo_estimate, occupation_ai_exposure, occupation_projection, occupation_task, research_source. Written by ingestion scripts, read by FastAPI. Supabase free tier: 500MB, sufficient for seeded BLS/Pew/O*NET data.
Vector Store Supabase · pgvector research_chunk table: chunked research text (~500 tokens, 100-token overlap) with 1024-dim vectors from Voyage AI voyage-3 model. Cosine similarity search at query time. Same Supabase instance as structured DB — no separate vector DB needed at this scale.
Embeddings Voyage AI · voyage-3 Called by ingestion scripts only — not at query time. Converts research text chunks to 1024-dim float vectors for pgvector storage. Free tier available for low volume. Voyage-3 chosen for strong performance on retrieval tasks.
Observability Sentry Single dashboard for frontend + backend. Browser SDK captures JS errors and Web Vitals from Vercel. Python SDK on Railway captures endpoint latency, Claude API call duration, rate limit hit rate, and unhandled exceptions. Free tier covers one project.
Occupation Data BLS OEWS 2024 + OOH Pre-loaded CSVs: wages (p10/p25/median/p75/p90), employment counts, 10-year projections, growth categories. Seeded into Postgres at deploy time via ingestion scripts. National + state + metro geo levels.
AI Exposure + Tasks Pew Research 2023 · O*NET Pew AI exposure scores (0–1) for 800+ occupations and per-task automation classifications. O*NET task descriptions and importance scores. Pre-loaded alongside BLS data. Joined by SOC code at query time.
Typical Request Flow
Step Actor Action
1 · Auth Browser → GitHub User clicks "Continue with GitHub." OAuth redirects to GitHub, returns authorization code. Auth.js exchanges for access token, reads username (read:user scope), persists JWT session cookie.
2 · Search Browser → Vercel User types occupation. Frontend queries /autocomplete with partial text. Vercel edge middleware checks IP rate limit. FastAPI queries soc_detailed_occupation for matching rows, returns SOC codes + labels.
3 · Analyze Vercel → Railway On selection, frontend POSTs {soc_detailed_occupation_id, label} to /analyze. FastAPI validates JWT, extracts GitHub user ID.
4 · Rate check Railway → Upstash FastAPI increments per-user counter in Upstash. If limit exceeded, returns 429 immediately. Otherwise continues.
5a · Data fetch Railway → Supabase FastAPI queries: occupation_geo_estimate (national wages), occupation_ai_exposure (Pew score + tier), occupation_projection (10-year outlook), occupation_task (automatable tasks). Single round-trip with joins.
5b · Vector search Railway → Supabase pgvector User's occupation label embedded via Voyage AI (or a cached query embedding). Top-k research_chunk rows retrieved by cosine similarity. research_source joined for citation metadata.
6 · Synthesis Railway → Claude Structured data + retrieved chunks assembled into a tool-call prompt. Claude synthesizes prose with inline citation markers [1], [2], [3]. Response streamed back to FastAPI.
7 · Render Vercel → Browser Results page renders: LLM prose with citation badges, AI exposure gauge (0–1 score + tier), wage distribution chart (p10–p90), employment projection, follow-up input, suggested chips.
8 · Follow-up Browser → Railway → Claude Follow-up questions sent with prior context + SOC data. Claude responds conversationally. Rate limit checked per follow-up. Not cached — per-session.
